Entwicklungsumgebung

wenn man mit PHP arbeitet, gibt es keinen anderen Weg als diese php Dokumente auf einen Webserver zu laden um sie auszuführen. für HTML/CSS wird kein Server zum Abrufen der Dateien Benötigt, da diese Clientseitig geladen werden deswegen benötigt man nicht mehr als ein Browser. Ich selbst benutze den Chrome Browser von Google, da die Entwicklertools sehr ausgereift sind. Um mit HTML, CSS und PHP zu programmieren/schreiben braucht man nicht unbedingt einen speziellen Editor, jedoch erleichtert ein Editor die mühsame Fehlersuche wesentlich. Deswegen benutze ich auch den Visual Studio Code von Microsoft denn es können auch nützliche Add-Ons installiert werden, welche beispielsweise die Tags automatisch schliessen. Sobald man den XAMPP Webserver heruntergeladen hat kann man mittels dem XAMPP Control Panel den Apache Dienst starten, und das Dashboard Seite von XAMPP über den localhost erreichen. Wenn man in der Browsersuchleiste localhost eingibt wird der htdocs Ordner nach Dateien durchsucht, welche er ausgeben kann. Im htdocs Ordner kann man weiter Unterordner erstellen und diese auch via "localhost/meinorder/datei.php" abrufen.

Formularfelder in HTML

Es gibt verschiedene Arten von Formularsteuerelementen, mit denen man Informationen des Benutzers der Website einholen kann. Es gibt unteranderem Texteingabe Felder, Auswahlfelder und Sendeschaltflächen um die Formulare einzureichen. Eine solche Übertragung auf den Webserver funktioniert folgendermassen: 1. Der Benutzer trägt die Werte ins Formular ein, 2. Er klickt auf die Sendeschaltfläche, in diesem Moment werden die einzelnen Steuerelementen mit dem vom Benutzer eigegebenen oder ausgewählten Werten an den Webserver gesendet, 3. Der Webserver verarbeitet diese Informationen mit einer Programmiersprache wie PHP oder C# verarbeitet, unter Umständen werden die Werte in der Datenbank eingetragen, 4. Der Webserver leitet den Benutzer aufgrund der empfangenen Werte auf eine neue Seite weiter.
